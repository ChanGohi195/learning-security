# PKI業界研究 - 信頼の基盤とその脆弱性

## はじめに

PKI（Public Key Infrastructure）は、インターネット上の「信頼」を技術的に担保する仕組みである。HTTPSの鍵マーク、ソフトウェアの署名、電子メールの暗号化——これらすべてがPKIに依存している。

しかし、この「信頼の基盤」は、純粋に技術的なものではない。数学的に堅牢な暗号アルゴリズムの上に、人間の組織、政治、経済が積み重なっている。そしてセキュリティは常に、最も弱い環節で破られる。

本稿では、PKI業界の構造、ビジネスモデル、そして歴史的なスキャンダルを通じて、「信頼とは何か」を問う。

---

## 1. 業界構造 - 寡占と権力の集中

### 市場規模

PKI市場は急成長を続けている。

| 年 | 市場規模 |
|----|----------|
| 2024 | 61.3億ドル |
| 2025 | 74.2億ドル |
| 2033（予測） | 342億ドル |

年平均成長率（CAGR）は21%。デジタル化の進展、IoTデバイスの爆発的増加、そして規制強化が成長を牽引している。

### Tier構造

市場は明確な階層構造を持つ。

**Tier 1（市場シェア45-50%）**

| 企業 | 特徴 |
|------|------|
| DigiCert | Fortune 500の80-90%にサービス提供。年商3.3億ドル |
| GlobalSign | クラウドベースPKI。数百万のデジタルID管理 |
| Entrust | 量子暗号に注力。2024年に信頼失墜、2025年に公開証明書事業を売却 |

**Tier 2（市場シェア15-20%）**

| 企業 | 特徴 |
|------|------|
| Sectigo | 2025年にEntrust事業を買収。Fortune 1000の36%が利用 |
| Thales | 特定業界向けPKIソリューション |

**無料CA**

| 企業 | 特徴 |
|------|------|
| Let's Encrypt | 非営利。3.6億ドメインにサービス提供。上位100万サイトの59.9%が利用 |

### 権力の所在

PKIにおける真の権力は、「信頼リスト」を決定する者にある。

```
ブラウザベンダー（Google, Mozilla, Apple, Microsoft）
    ↓
「このCAを信頼する/しない」を決定
    ↓
CAの生殺与奪を握る
```

2024年、EntrustはGoogle・Apple・Mozillaから信頼を剥奪され、事業売却に追い込まれた。技術的には何も変わっていない。「信頼しない」という決定だけで、企業は終わる。

## 2. ビジネスモデル - 信頼の商品化

### 二極化する市場

PKI市場は「無料」と「有料」に二極化している。

| 観点 | Let's Encrypt | 商用CA（DigiCert等） |
|------|---------------|---------------------|
| 価格 | 無料 | 年間数百〜数千ドル |
| 証明書タイプ | DV（ドメイン検証）のみ | DV, OV, EV, コード署名, S/MIME |
| 有効期限 | 90日 | 1-2年（→2029年に47日へ短縮） |
| サポート | なし | 24/7専門サポート |
| 保証 | なし | 損害賠償保証付き |
| ターゲット | 個人、小規模事業者 | 大企業、金融、政府 |

### Let's Encryptの革命

Let's Encryptは2015年に設立され、HTTPSを民主化した。

- 運営：ISRG（Internet Security Research Group）、非営利501(c)(3)
- 収益：100%寄付・スポンサー（累計1,700万ドル以上）
- 主要スポンサー：Mozilla, EFF, AWS, Google, Cisco, Gates Foundation

彼らが証明したのは、「暗号化は商品ではなく、インフラである」という思想だ。

### 商用CAの差別化

無料CAの台頭に対し、商用CAは「証明書単体」から「プラットフォーム」へシフトしている。

DigiCertの例：
- DigiCert ONE：統合PKI管理プラットフォーム
- 2024-2025年、証明書顧客の67%がONEを追加購入
- ARR 100万ドル以上の顧客が年30%増加

「信頼を売る」から「信頼の管理を売る」への転換である。

## 3. スキャンダル史 - 失敗の教科書

PKI業界の歴史は、失敗の歴史でもある。そして、その失敗から学ぶことこそがセキュリティの本質だ。

### DigiNotar（2011年）- 業界の転換点

**何が起きたか**

オランダの認証局DigiNotarが、2011年6月から侵害されていた。

- 531の不正証明書が発行された（344ドメイン）
- 対象：google.com, microsoft.com, mozilla.org, yahoo.com, skype.com, torproject.org...
- 攻撃元：イラン・テヘランのIPアドレス

**杜撰な管理**

侵入調査で判明した実態：

- 全8台のCA管理サーバーが侵害されていた
- 30件の重要セキュリティ更新が未適用
- アンチウイルス未導入
- ログ不備で侵害の全容が把握不能

**被害**

イラン国内で約30万のGmailアカウントがMITM（中間者）攻撃の標的となった。

```
イラン国民 → 偽のgoogle.com証明書 → 政府の監視サーバー → 本物のGoogle
```

通信内容は全て傍受可能な状態だった。

**結末**

- 2011年8月29日：全主要ブラウザが信頼削除
- 2011年9月：DigiNotar破産

**教訓**

1件の侵害で、企業は終わる。PKI企業には「失敗が許されない」という宿命がある。

### CNNIC（2015年）- 国家の影

**何が起きたか**

中国のCNNIC（中国互聯網絡信息中心）傘下の中間CAが、google.comの不正証明書を発行した。

**背景**

CNNICは中国政府傘下の組織である。

```
中国政府
    ↓
CNNIC（ルートCA）
    ↓
MCS Holdings（中間CA）
    ↓
google.comの偽証明書発行
```

**Googleの対応**

- Chromeから CNNIC証明書を段階的に削除
- 2015年4月、完全削除

**教訓**

国家がPKIを監視インフラとして利用する可能性は、理論ではなく現実である。

### Symantec（2017年）- 巨人の転落

**何が起きたか**

世界最大級のCAであったSymantecが、3万件以上の不正証明書を発行していたことが発覚。

- 内部監査の欠如
- 証明書発行プロセスの違反
- EV証明書の検証不備

**結末**

- Googleが段階的な信頼削除を発表
- SymantecはCA事業をDigiCertに売却

市場シェアトップでも、信頼を失えば終わる。

### Entrust（2024年）- 現在進行形

**何が起きたか**

CA/Browser Forumの要件に非準拠であることが発覚。

- 証明書発行プロセスの不備
- 透明性要件の違反

**結末**

- 2024年6月：Google, Apple, MozillaがEntrust証明書の不信化を発表
- 2024年11月：ブラウザ信頼停止
- 2025年1月：公開証明書事業をSectigoに売却

Tier 1企業でさえ、規制違反で市場から退場する。

## 4. 国家とPKI - 監視インフラとしての悪用

### 証拠が示す事実

DigiNotar攻撃の詳細な分析から、以下が判明している：

- 攻撃元IPはイラン・テヘラン
- 攻撃者は「Comodohacker」を名乗り、イラン愛国的声明を発表
- イラン革命防衛隊（IRGC）の関与が濃厚

CNNICの事例では：

- 中国政府傘下の組織が不正証明書を発行
- 中国国内でのMITM監視に使用された可能性

### 構造的な脆弱性

PKIの構造は、国家による悪用を許容する設計になっている。

```
問題1：どの国のCAでも、どのドメインの証明書も発行できる
    ↓
中国のCAがgoogle.comの証明書を発行可能

問題2：ルートストアへの組み込みは各国の判断
    ↓
政府がブラウザベンダーに圧力をかけられる

問題3：ユーザーは証明書の発行元を意識しない
    ↓
偽証明書に気づけない
```

### 数字で見る現実

- 34カ国がサイバー作戦を国家として後援
- 中国・ロシア・イラン・北朝鮮で全体の77%
- CA侵害は、国家レベルの監視インフラとして機能

## 5. 業界の対応 - 透明性と自動化

スキャンダルの連続を受けて、業界は構造的な改善を進めてきた。

### Certificate Transparency（CT）

2011年のDigiNotar/Comodo侵害を受け、Googleが提唱。

**仕組み**

```
CA：証明書を発行
    ↓
公開ログに記録（誰でも閲覧可能）
    ↓
不正発行を誰でも検知可能
```

**効果**

- 2018年4月：Chromeが全新規証明書にCT義務化
- 「秘密の取引」から「公開イベント」への転換
- 検出可能性の劇的向上

**限界**

発見しても、失効（Revocation）が機能しにくい問題は残る。

### 証明書寿命の短縮

CA/Browser Forumは証明書の有効期限を段階的に短縮する方針を決定した。

| 時期 | 最大有効期限 |
|------|-------------|
| 現在 | 398日 |
| 2026年3月 | 200日 |
| 2027年3月 | 100日 |
| 2029年3月 | 47日 |

**意図**

- 漏洩した鍵の影響を最小化
- 更新頻度向上でセキュリティ強化
- 自動化の強制（年8回更新は手動では不可能）

### ACME自動化

Let's Encryptが開発したACMEプロトコルが標準化（RFC 8555）。

- 証明書の発行・更新・失効を完全自動化
- DevOpsツール（Kubernetes, Ansible, Terraform）に統合
- 47日証明書時代の必須インフラ

## 6. 残された課題 - 量子時代への備え

### 量子コンピュータの脅威

現在のPKI暗号（RSA, ECDSA）は、量子コンピュータで解読可能になる。

**Shor's Algorithm**

```
量子コンピュータ
    ↓
素因数分解が高速に解ける
    ↓
RSAの「計算量の非対称性」が崩壊
```

**"Harvest now, decrypt later"**

```
攻撃者：暗号化された通信を今のうちに保存
    ↓
数年後：量子コンピュータで解読
```

長期保存価値のあるデータ（医療記録、国家機密、企業秘密）は、既に脅威にさらされている。

### ポスト量子暗号（PQC）

NIST は2024年8月にPQC標準を発表した。

| アルゴリズム | 用途 |
|-------------|------|
| ML-DSA（旧Dilithium） | デジタル署名 |
| ML-KEM（旧KYBER） | 鍵交換 |
| SPHINCS+ | 署名（代替） |

**移行スケジュール**

- 2030年まで：RSA/ECCからPQCへ移行推奨
- 2035年以降：米国政府機関でPQC必須

**課題**

- 後方互換性：旧システムとの共存
- 証明書サイズ：PQC署名は従来より大きい
- 移行規模：平均的組織は25万証明書を保有

## 7. 学習者への教訓

### 技術と信頼の境界

```
暗号アルゴリズム → 数学が保証
    ↓
PKI → 人間・組織・法律が保証
    ↓
最も弱いところで破られる
```

セキュリティは技術だけでは完結しない。

### 攻撃者の視点

PKIを攻撃するなら：

1. **CAを侵害する** — DigiNotarのように
2. **国家権力を使う** — CNNICのように
3. **規制違反を突く** — Entrustのように
4. **量子時代を待つ** — "Harvest now, decrypt later"

### 防御者の視点

PKIを守るなら：

1. **Certificate Transparencyを監視する** — 不正発行の検知
2. **CAAレコードを設定する** — 許可CAを制限
3. **自動化を導入する** — ACME、47日時代への備え
4. **PQC移行を計画する** — 暗号棚卸し、NIST標準評価

### 本質

PKIは数学の上に人間が乗っている。そして人間は、常に最も弱い環節である。
